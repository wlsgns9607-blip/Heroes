# 🔥 히어로즈 (Heroes) — 앱 기획 프레젠테이션 웹사이트

소방관을 위한 **감사 & 리스펙 플랫폼, 히어로즈**의 기획서를 한 페이지 웹 프레젠테이션으로 구성한 정적 사이트입니다.
와이어프레임 프롬프트, 디자이너·개발자 협업 핸드오프 내용까지 이 문서 하나에 정리되어 있습니다.

---

```
heroes-app/
├── index.html   ← 전체 페이지 마크업 (6개 섹션 + #wireframe 앱 와이어프레임 섹션)

├── script.js    ← 스크롤 reveal 애니메이션, 네비 하이라이트
└── README.md    ← 이 파일
```

- 현재 보이는 섹션에 따라 `nav-links a.active` 자동 하이라이트

## 🎙️ 기능 04 — 음성 인식 출동 기록 입력

> 현장에서 손을 쓸 수 없는 소방관이 목소리만으로 출동 기록을 완성하는 핸즈프리 입력 시스템.

### 배경 및 필요성

소방관은 출동 직후 장갑을 낀 채, 연기와 소음이 가득한 현장에서 출동 데이터를 입력해야 합니다. 기존 드롭다운 Form은 두 손이 자유로운 상황을 전제하고 있어, 실제 현장에서 사용하기 어렵습니다. 음성 인식은 이 물리적 제약을 제거하는 가장 직접적인 해결책입니다.

### 동작 방식

1. **마이크 버튼 탭** — 화면 하단 고정 마이크 FAB(Floating Action Button)을 한 번 탭해 녹음 시작.
2. **음성 발화** — "건축물 화재, 인명 구조 2명, 진압 완료" 형식으로 말하면 됩니다.
3. **실시간 transcript** — Web Speech API가 음성을 즉시 텍스트로 변환해 화면에 미리보기로 표시.
4. **필드 자동 매핑** — 출동 유형 / 인원 수 / 결과 / 특이사항 필드에 파싱된 값이 자동으로 채워집니다.
5. **확인 후 저장** — 대원이 눈으로 내용을 검토하고 "저장" 버튼 한 번으로 기록 완료.

### 핵심 기술 스펙

| 항목 | 내용 |
|------|------|
| **음성 인식 엔진** | Web Speech API (`SpeechRecognition`) — 브라우저 네이티브, 별도 서버 불필요 |
| **언어 설정** | `lang: 'ko-KR'` 한국어 특화 모델 |
| **커스텀 사전** | 소방 도메인 용어(진압, 구조, 구급, 잔화정리, 선착대 등) Grammar 등록으로 인식률 향상 |
| **노이즈 처리** | 현장 소음 환경 대응을 위한 음성 전처리 필터 적용 |
| **폴백 모드** | 음성 인식 실패 또는 지원 안 되는 브라우저 시 기존 드롭다운 Form으로 즉시 전환 |
| **보안** | 음성 데이터는 서버에 저장하지 않고 클라이언트에서만 처리 후 텍스트 결과만 서버 전송 |

### UX 고려사항

- 장갑을 낀 상태에서도 쉽게 탭할 수 있도록 마이크 버튼 크기를 최소 **56px × 56px** 이상으로 설계
- 녹음 중에는 버튼이 빨간색으로 맥박처럼 깜빡이는 pulse 애니메이션으로 상태 표시
- 인식 결과가 의도와 다를 경우 필드를 개별적으로 수동 수정 가능
- 오프라인 환경(현장 통신 불량)에서는 폴백 Form이 자동 활성화되고 배너로 안내

# 🧩 와이어프레임 & 협업 핸드오프

> 아래는 디자이너·개발자와의 협업을 위한 와이어프레임 프롬프트 및 핸드오프 자료입니다.

## 섹션별 와이어프레임 프롬프트

> Figma AI / v0.dev / Midjourney(레이아웃 스케치용) 등에 그대로 붙여넣어 사용할 수 있도록 작성했습니다.

### 🅰 Hero 섹션 (`#hero`)
```
다크 테마(#0d0d0d 배경) 히어로 섹션 와이어프레임.
좌측: 작은 eyebrow 텍스트(빨강, 대문자, 자간 넓음) → 대형 제목(3줄, 한글 디스플레이 폰트,
그 중 한 단어는 레드 강조) → 서브 설명 1~2줄(회색, 골드 강조 단어 포함) →
가로로 나열된 4개의 pill 배지(테두리형, 아이콘+텍스트).
우측: 배경 뒤로 거대한 반투명 워터마크 텍스트("HEROES") + 전경에 3개의 불꽃 형태
그래픽 요소(레드→골드 그라데이션, 크기 다른 3개가 겹쳐진 형태).
상단 고정 내비게이션 바(로고 좌측, 메뉴 4개 우측, 반투명 블러 배경).
모바일 버전: 불꽃 요소 제거, 내비 메뉴 숨김, 텍스트 중앙 정렬 스택.
```

### 🅱 Problem 섹션 (`#problem`)
```
3열 카드 그리드 와이어프레임(다크 배경, 카드형 컴포넌트).
각 카드: 상단 이모지 아이콘(2rem) → 카드 제목(굵은 글씨) → 2~3줄 설명 텍스트.
카드는 미묘한 테두리, hover 시 위로 4px 이동 + 테두리 레드 강조.
섹션 상단에 작은 eyebrow + 2줄짜리 큰 섹션 타이틀(한 단어 레드 강조).
모바일: 3열 → 1열 스택.
```

### 🅲 Persona 섹션 (`#persona`)
```
3개의 인물 페르소나 카드 와이어프레임, 배경은 살짝 밝은 서페이스 톤.
각 카드 상단: 원형 아바타(이니셜, 그라데이션 배경) + 이름 + 역할/나이/소속 텍스트.
본문: "가장 불편한 것"(레드 pill 태그) + 설명 1~2줄,
"원하는 것"(골드 pill 태그) + 설명 1~2줄.
카드 하단: 인용구 블록(좌측 레드 보더, 이탤릭, 옅은 배경) — 카드 맨 아래 고정(margin-top:auto).
3열 그리드, 모바일 1열.
```

### 🅳 Features 섹션 (`#features`)
```
세로로 나열된 4개의 기능 리스트 아이템 와이어프레임(가로 구분선으로 분리).
각 행: 좌측 큰 번호(01~04, 옅은 레드, 디스플레이 폰트) — 중앙 제목+설명 —
우측 아이콘(대형 이모지 스타일).
4번째 항목만 특별 강조: 배경에 은은한 레드/골드 그라데이션, 우측 상단에 "NEW" 뱃지,
설명 문단이 4개로 길게 분할(리드 문장 강조 + 상세 3문단 + 기술 스택 노트 라인).
모바일: 3열 그리드(번호/본문/아이콘) → 2열로 축소, 아이콘 숨김.
```

### 🅴 Impact 섹션 (`#impact`)
```
3개의 통계 카드(중앙 정렬, 큰 숫자 강조) + 하단 "다음 버전 계획" 배너 와이어프레임.
통계 카드: 초대형 숫자(레드, 단위는 골드로 작게) → 설명 텍스트 1~2줄, 카드 hover 시 상승.
하단 배너: 전체 폭, 은은한 레드/골드 그라데이션 배경, 제목(로켓 이모지+골드 텍스트) +
본문 설명 단락.
3열 → 모바일 1열.
```

### 🅵 Footer
```
중앙 정렬 푸터 와이어프레임. 로고(골드) → 서브 텍스트(플랫폼 한줄 소개) →
작은 크레딧 텍스트(개발자/타겟 정보). 상단 구분선.
```

### 🅶 신규 기능 상세 프롬프트 — 음성 인식 출동 기록 입력 (별도 화면 설계용)
```
모바일 앱 화면 와이어프레임, 다크 테마.
하단 중앙에 고정된 원형 마이크 FAB 버튼(56x56px 이상, 레드).
녹음 중 상태: 버튼이 펄스 애니메이션으로 깜빡임 + 상단에 실시간 transcript

그 아래: 출동 유형 / 인원 수 / 결과 / 특이사항 4개 필드가 음성 인식 결과로
자동 채워진 상태의 카드형 폼, 각 필드는 수동 수정 가능한 편집 아이콘 포함.
화면 하단: "저장" 버튼(레드, 풀폭).
오프라인/인식 실패 시 대체 화면: 상단에 경고 배너("음성 인식 실패 — 드롭다운
```

## 실제 구현된 앱 와이어프레임 (`index.html` 내 `#wireframe` 섹션)

> 위 프롬프트 기반 목업과 별개로, **기존 `index.html`에 실제 와이어프레임을 직접 추가 구현**했습니다.
> 웹 프레젠테이션이 아니라 **모바일 앱**으로 개발할 예정이므로, 프레임 크기를 대중적인 스마트폰 규격에 맞췄습니다.

### 프레임 규격
| 항목 | 값 |
|---|---|
| 기준 프레임 | iPhone 14/15 — **390×844** (대중 보급률 1위 규격) |
| 보조 대응 | Galaxy S 시리즈 — 360×800 (동일 비율로 자동 스케일) |
| 구현 방식 | `aspect-ratio: 390/844` + `max-width: 240px` 카드형 프레임, 노치(`.phone-notch`) 포함 |
| 위치 | `index.html` — "기대 효과" 섹션과 Footer 사이, `id="wireframe"` |
| 내비게이션 | 상단 메뉴에 `와이어프레임` 링크 추가 완료 |

### 구현된 4개 화면
| # | 화면 | 주요 와이어프레임 요소 |
|---|---|---|
| 01 | 출동 데이터 입력 Form | 상단바(뒤로가기+타이틀) → 드롭다운 필드 3종(출동유형/인원/결과) → 특이사항 textarea → 하단 고정 "저장하기" CTA |
| 02 | 명예 배지 시스템 | 상단 자부심 게이지(프로그레스바, 레드→골드 그라데이션) → 3×2 배지 원형 그리드(획득/미획득 opacity 구분) → 최근 획득 리스트 |
| 03 | 시민 감사 & 리스펙 댓글 | 아바타+텍스트 라인으로 구성된 댓글 카드 3개 → 하단 고정 입력창 + 커피 쿠폰 아이콘 |
| 04 | 음성 인식 출동 기록 입력 | 실시간 transcript 박스(골드 대시 보더, "실시간 인식 중…") → 자동 매핑된 필드(체크마크로 자동완성 표시) → 하단 중앙 마이크 FAB(글로우 효과) |

## 개발자 핸드오프 체크리스트

### 현재 구현된 인터랙션 로직
| 기능 | 구현 방식 |
|---|---|
| 스크롤 reveal | `IntersectionObserver`(threshold 0.12), 같은 부모 내 형제 요소 90ms 스태거 딜레이 |
| 활성 내비 하이라이트 | `IntersectionObserver`(rootMargin `-40% 0px -55% 0px`)로 현재 섹션 감지 → `.active` 클래스 토글 |
| 불꽃 애니메이션 | 순수 CSS `@keyframes flicker`, 3개 레이어 비동기 `animation-delay` |

### 신규 기능(음성 인식) 구현 시 필요한 기술 스펙
| 항목 | 스펙 |
|---|---|
| 엔진 | Web Speech API (`SpeechRecognition`), 브라우저 네이티브, 서버 불필요 |
| 언어 | `lang: 'ko-KR'` |
| 커스텀 사전 | 소방 도메인 용어(진압/구조/구급/잔화정리/선착대 등) Grammar 등록 |
| 노이즈 처리 | 현장 소음 대응 전처리 필터 |
| 폴백 | 미지원/실패 시 기존 드롭다운 Form으로 즉시 전환 + 배너 안내 |
| 보안 | 음성 데이터는 클라이언트 처리만, 서버에는 텍스트 결과만 전송 |
| UX 제약 | 마이크 버튼 최소 56×56px, 녹음 중 pulse 애니메이션, 필드별 수동 수정 가능 |

### 개발 우선순위 제안
1. 기존 정적 페이지 → 컴포넌트 단위 리팩터링 (프레임워크 도입 시)
2. 실제 데이터 입력 Form(기능 01) MVP 구현 — 10대 출동 유형 드롭다운
3. 명예 배지 시스템(기능 02) — 배지 지급 로직 + 시각화
4. 시민 감사 댓글 UI(기능 03)
5. 음성 인식 입력(기능 04) — Web Speech API PoC → 폴백 로직 → 필드 자동 매핑
